1. Foreword

This program is an editor for Commodore 64 sprites. I have written it as my small competition for free version of SpritePad and addressing a few drawbacks of already mentioned SpritePad's free version. It is not my intention to dethrone SpritePad, especially its commercial version should be better choice anyway.

Please let me introduce these drawbacks and my solutions implemented in "Master of Sprites". The main problems are: lack of batch mode support, poor support for animation with sprite overlay or lack of support for displaying connected multisprites. There are also some minor problems like portability issues on operating systems different than Windows or or relatively old-fashioned and unhandy user interface.

The problems with portability are maybe not so big and important - SpritePad runs perfectly well on Wine after all. However personally I feel it's always good and satisfying to have similar program made in Java. In other hand I didn't made anything much better in the topic of user interface. I feel fine with overall look & feel of "Master of Sprites" but everything is pretty standard as for Java Swing application. There are many drawbacks too.

The already mentioned main problems was my biggest motivation to make this program. Master of Sprites is capable to export its project from command line interface. The preview tab of its main window contains nice animation preview with support for both overlay and multisprites. The "Overlay Distance" setting makes easy to "arrange" memory bank in the way which takes into account both animation and overlay. If you check "Overlay Next Sprite" option then current sprite will automatically overlay the sprite, further by "Overlay Distance", in the memory bank. It is just that simple.

If some buttons or options of this program are mysterious thing for you, please take a look at "Quick Guide to the GUI" chapter. If you want to know how to export files from command line then go directly to "Command Line Parameters". I think this is enough for an introduction.

3. Command Line Parameters

Opening Project From Command Line

java -jar MasterofSprites.jar project_file

With the command above you can open some project file from the command line interface. Unfortunally it doesn't support opening multiple windows with multiple projects.

Export From Command Line

Master of Sprites offers ability to export its projects from command line as well from shell scripts or batch files. Syntax of appropriate command is showed below:

java -jar MasterofSprites.jar -e [format] project_file [-o output_file]

Avaible export formats are:

  • Raw binary - default export format. If you leave the format argument empty you will export your project as the raw binary.
  • PRG file - if you want to export to a PRG file then specify load address of your output PRG file as the hexadecimal number prefixed with "0x".
  • Assembly source code - there is possible to export projects to source code of these three popular assemblers: ACME, KickAss and TMPx. Just replace the format argument with the name of one of these three assemblers. Names are case insensitive here, so both ACME or acme or even aCMe are valid names.

If you do not specify output file then Master of Sprites will just change extension of your project file to raw, prg or asm.

Unfortunally currenty is not possible to make multiple exports in one command by using wildcards like '*' or '?'. On UNIX-like operating systems you can do the same job by looping in the shell script or by using capabilities of makefiles. Worse if you are using Windows batch files - as far as I know there is no looping there. :(
