1. File Menu

Functionality of this menu is rather pretty obvious, right? So let me, to just explain, a few strange decisions I had made during work on the Master of Sprites. These are maybe kinda stinky excuses but at least you will not wonder why save menu item is always enabled etc.

First of all, you probably see the project work time in bottom right corner of window. I had added this feature mainly for fun, but personally I very like to see the time I have spend on some particular work. This time, for reason of simplicity and portability, is saved inside project file. So it is the reason why I have left save menu item always enabled - to let you all cheat and save >2hr work time when you was really watching a soap opera instead of working. Seriously - sometimes there is a need to save the timestamp even if no changes have been made.

Second matter. On the titlebar, sometimes you can see an indicator of unsaved file, the tilde ~. As you probably expect changing background color shall make project unsaved, right? Yes, it makes. But this kinda change, unfortunelly is not noticed by undo/redo history. But lets leave, all details of this topic, for the second section of manual 'Edit Menu'.

Third matter. The lack of a new file dialog. I have decided to do not bother the user by settings of memory bank size, default sprite color etc. Anyway, these all are very easy to change later. See the section of 'Tips & Tricks' if you need a crash course in this topic.

Ok... Thats maybe all for this section. Remember that sometimes one matter is very close to the other one and necessary read my whole rant. Good luck!

2. Edit Menu

Lets divide this section into three smaller ones.

Undo and Redo History

As I previously mentioned (see the 'File Menu' section) - not every change in project is noticed by the undo/redo history. Generally, Master of Sprites push to the history only changes which regards particular sprite or sprites. So we do not push such silly changes like changing background or one of multicolor mode colors. These are far to easy to reverse, anyway.

Rest is pretty typical and default - CTRL-Z to undo, CTRL-Y to redo, nothing to specially explain as I think. BTW: History is implemented in a very simple way - it simply makes copies of whole memory bank. It is not so huge bulk of data for modern computers, however if you want to edit whole 64kB of data (1024 sprites) - probably you will encounter the noticable slowness.

Cut/Copy/Paste

Yes again, pretty obvious topic. But let me shine a bit and mark up that Master of Sprites can run multiple instances of itself and you can easily copy and paste between them. Just remember to select something first to cut or copy it. And to copy something first to paste it.

Special Paste Modes: AND, OR and XOR

Now, something what teddy bears like the most. Do you understand what are bitwise operators (AND/OR/XOR)? If not then it is somewhat hard to explain shortly - please take a look at google or wikipedia. If yes then you probably understand well how these three are working.

Let me just explain everybody, that these paste modes, are very useful when you want to make some change on multiple sprites simultaneously. Just play around and try them on some wider selection.

3. Sprite Menu

In this menu you will find all possible actions regarding particular single sprite. Except same drawing maybe. Lets list them all and describe briefly:

  • Setting an unique name - this one is useful only when you want to export your project to assembly code. After you set a name to the sprite, Master of Sprites will add label with your name inside exported asm file. It can be helpful to do not mess around later, with nothing telling labels like sprite0 or sprite31.
  • Slide - Helpful to move a little your sprite if you did not fitted it perfectly to its frame.
  • Flip - Now you can create a second version of your sprite turned around in opposive direction!
  • Reflect - Helps to make perfectly symetrical sprites.
  • Rotation - Not always working perfectly (especially for multicolor mode effect can be not very accurate) but useful tool to rotate your sprite as needed. Important thing which you can do not notice - in the rotation dialog you can enter negative number to rotate in opposive direction (counter-clockwise).
  • Negation - Switching between negative and positive versions of your sprite. You know...

All actions in this menu affect only one sprite at the time. See the 'Selection Menu' section if you want to read about selection-wide actions.

5. Command Line Parameters

Opening Project From Command Line

You can automatically open project if you specify its name/path from command line:

java -jar MasterofSprites.jar project_file

It is sometimes useful because on most modern operating systems, then you can drag & drop project file into MasterofSprites.jar file, and Master of Sprites will automatically open your project.

Export From Command Line

Master of Sprites offers option to export its projects from command line. Syntax is following:

java -jar MasterofSprites.jar -e [format] project_file [-o output_file]

Avaible export formats are:

  • Raw format - leave format argument empty and Master of Sprites will export just same raw binary data of your sprites, without any additional informations.
  • PRG file - specify load address of prg file as the format argument. Load address should be a hexadecimal number prefixed by '$' or '0x'.
  • Assembler source code - specify name of assembler and Master of Sprites will write code compatible with syntax of specified assembler. Possible Choices are: kickass, acme or tmpx.

If you do not specify output file then Master of Sprites will just change extension of your project file to raw, prg or asm.
